Project 2

Analytical Applications LLC. Project

This is a company created by me to produce a product and employment for myself during the COVID-19 lockdown. This is still a work in progress and now a side project for me as time allows. While many functions are still in the prototype phase I have had some field instillation success.

This project was to cross abilities of PLC's and edge computers to allow IOT and industrial applications to mix functionality. As you use higher level function in a PLC it losses some of its ability because it has very limited resources. If connected with an overlay of computing it is allowed to retain its reliability and speed. This is because tasks like calibration, tunning, data logging and some inferred or long term calculational alarming can be done externally. Analytics such as pump wear over time for maintenance or battery charging rate and capacity are examples best left to computers. This becomes ever harder as the applications become more and more remote or standalone.

One such applications allowed remote access into a PLC in the oil field using a JavaScript and HTML interface, a Python computation backbone, and C++ communications with the PLC. These programs use ini, xml and Json objects to allow live data for a internal site webpage. This provides remote access and data collection.

These data communications came in the form of OPC-UA, RS-485, CIP, and ProfiNet protocols. When these communications styles were not available or not allowed by the customer to tie into the system, it was set up through 16-bit or 32-bit UDP messaging. This granted less control but provided third party defensibility.
